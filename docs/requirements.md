# Requisitos - Roadmap MVP

## Objetivo
Crear un servicio para gestionar roadmaps (títulos y descripciones) expuesto por una API REST y consumible desde un frontend Angular.

## Requisitos funcionales
- RF1: Crear un roadmap (titulo, descripción).
- RF2: Listar roadmaps.
- RF3: Consultar roadmap por id.

## Requisitos no funcionales
- RNF1: Arquitectura hexagonal (separación dominio / aplicación / adaptadores).
- RNF2: Tests unitarios para casos de uso del dominio.
- RNF3: Documentación básica de análisis y diseño.

## Alcance MVP
Implementación backend con repositorio en memoria y adaptador REST; frontend es cliente Angular mínimo.
